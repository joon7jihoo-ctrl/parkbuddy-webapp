# ParkBuddy Web 평가분석보고서

작성일: 2026-06-05

## 1. 평가 범위

이번 평가는 개선 포인트와 계획된 MVP 개발 항목의 구현 완료 여부, 주요 사용자 흐름의 정상 동작 여부, 배포 안정성을 대상으로 수행했습니다.

평가 대상 기능:

- 회원 등록/수정/삭제/검색, CSV 일괄등록
- 동호회 이름/직책 기반 회원 정보 관리
- 사용자 구장 등록/삭제와 라운딩 생성 연동
- 홀별 규정타수 입력 및 점수 입력 기본값 반영
- 신페리오 숨김 홀 자동 선정, 직접 선택, 공개 여부 설정
- 스트로크 플레이, 신페리오, 매치 플레이, 스크램블, 포섬, 포볼, 스테이블포드 순위 계산
- 조편성, 팀전/페어전 점수 입력, 기록 저장
- 라운딩 기록 상세 보기, CSV 내보내기, 삭제
- 회원 목록 접힘/확장 표시
- 화면 이동 시 최상단 스크롤, 브라우저 뒤로가기 이동
- 개인점수관리와 기준파 대비 기록 추이 그래프
- Supabase 자동 저장
- GitHub Pages/로컬 Vite 실행 안정성

## 2. 구현 완료 평가

기능 구현 상태는 MVP 기준으로 완료로 평가합니다.

- 회원 관리: 검색, 소속 동호회, 직책, 연락처 형식, CSV 인코딩/파싱 안정성이 반영되었습니다.
- 경기 운영: 라운딩 생성부터 참가자 선택, 조편성, 점수 입력, 순위표, 기록 저장까지 단일 흐름으로 동작합니다.
- 경기 방식: 개인전, 팀전, 페어전, 포인트 기반 경기 방식을 모두 같은 순위 엔진에서 처리합니다.
- 신페리오: 홀 규모별 2/3 숨김 홀 원칙을 9홀 블록 단위로 적용하고, 파 합계도 전체 기준파의 2/3에 맞추도록 설계되었습니다.
- 기록 관리: 저장된 기록의 상세 확인, 삭제, CSV 내보내기가 가능해졌습니다.
- 화면 UX: 메인 Supabase 상태 표시를 제거하고, 화면 이동 시 최상단이 보이며, 브라우저 뒤로가기가 이전 화면을 복원합니다.
- 회원 목록: 기본 표시 정보를 이름/직책으로 줄이고, 선택 시 상세 정보와 수정/삭제 동작을 확장 표시합니다.
- 개인점수관리: 저장된 기록에서 회원별 기준파 대비 점수 추이를 그래프로 확인할 수 있습니다.
- 저장 방식: Supabase 단일 JSON 상태 저장에 회원, 기록, 최근 장소, 사용자 구장을 포함했습니다.

## 4. 품질 분석

강점:

- 하나의 점수 입력/순위 계산 구조로 개인전과 팀전을 함께 처리해 확장성이 좋아졌습니다.
- 신페리오 숨김 홀 선정은 9홀 단위 배분, 숨김 홀 수, 파 합계 기준을 함께 고려합니다.
- Supabase 저장 대상을 확장하면서도 기존 단일 상태 저장 구조를 유지해 변경 위험을 줄였습니다.
- 자동 브라우저 스크립트가 전체 핵심 흐름을 반복 검증할 수 있게 되었습니다.
- 개인점수 그래프는 기준파 대비 타수를 사용해 9/18/27/36홀 기록이 섞여도 비교가 가능합니다.
- 회원 목록은 접힘/확장 구조로 바뀌어 대량 회원 목록에서 화면 밀도가 개선되었습니다.

제한 사항:

- 로그인과 권한 관리는 아직 구현되지 않았습니다.
- Supabase는 MVP 단계의 단일 JSON 상태 저장 방식이므로, 다중 동호회 운영이 본격화되면 정규 테이블과 RLS 정책으로 분리해야 합니다.
- 모든 참가자의 점수가 동일하면 공동 순위로 표시됩니다. 운영 규칙상 순위를 강제로 나눠야 한다면 추가 타이브레이커가 필요합니다.
- CSV 내보내기는 기록 단위 수동 다운로드 방식이며, 기간별 통계 리포트는 아직 없습니다.
- 개인점수 그래프는 현재 저장된 라운딩 기록 기반이며, 기간 필터나 경기 방식별 필터는 아직 없습니다.
- 브라우저 주소는 화면별 고유 URL이 아니므로 새로고침 후 특정 상세 화면으로 바로 복원되지는 않습니다.
- 상단 홈/뒤로가기 버튼을 제거했기 때문에 앱 내 빠른 홈 이동은 홈 화면 메뉴와 브라우저 이동에 의존합니다.

## 5. 개선사항 도출

우선순위 높은 개선사항:

- 로그인과 동호회 권한 관리: 관리자/일반 사용자 권한을 분리하고, 일반 사용자는 본인이 가입한 동호회만 볼 수 있도록 Supabase Auth와 RLS를 도입해야 합니다.
- Supabase 정규 테이블 전환: 현재 단일 JSON 상태를 `clubs`, `members`, `rounds`, `round_results`, `memberships` 등으로 분리하면 검색, 권한, 통계 확장이 쉬워집니다.
- 개인점수관리 고도화: 기간별 필터, 경기 방식별 필터, 9/18홀 환산 평균, 최근 N경기 추세선, 회원 간 비교 그래프를 추가할 수 있습니다.
- 화면 URL 상태화: `#members`, `#records`, `#personalScores` 같은 라우트 또는 React Router를 도입하면 새로고침/공유/브라우저 이동 안정성이 좋아집니다.
- 기록 품질 관리: 점수 입력 누락, 비정상 타수, 경기 방식별 점수 범위 검증을 강화하면 운영 오류를 줄일 수 있습니다.
- 자동 테스트 확장: 현재 브라우저 흐름 검증 중심이므로, 순위 계산과 신페리오 숨김 홀 선정은 순수 함수 단위 테스트를 추가하는 것이 좋습니다.
- 모바일 UX 다듬기: 회원 수가 많을 때 검색/필터 고정, 하단 액션 고정, 그래프 가로 스크롤 안내를 보완하면 현장 사용성이 더 좋아집니다.

## 6. 결론

현재 ParkBuddy Web은 계획된 MVP 운영 흐름을 웹앱으로 수행할 수 있는 상태입니다. 회원 관리, 라운딩 생성, 조편성, 점수 입력, 순위 계산, 기록 관리, 개인점수 추이 확인, Supabase 저장, 자동 검증이 모두 연결되었습니다.

다음 단계의 우선순위는 로그인 도입, 동호회별 권한 분리, Supabase 정규 테이블 전환, 개인 통계 고도화입니다.

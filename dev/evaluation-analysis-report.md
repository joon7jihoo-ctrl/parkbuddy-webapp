# ParkBuddy Web 평가분석보고서

작성일: 2026-06-12

## 1. 평가 범위

이번 평가는 라운딩 기록 삭제 안정성, 홈 화면 반응형 문구, 다크모드 가독성, CSV 저장 UX, 라운딩 생성 단계/뒤로가기 흐름, 코스별 기준파 입력, 점수 입력 그리드, 공유 링크 점수 입력, 청백전 경기방식, 모바일 밀도, 모션 기반 전환 UX를 대상으로 수행했습니다.

검증 대상 화면:

- 홈
- 회원 등록 및 관리
- 라운딩 생성
- 참가자 선택
- 조편성 결과
- 라운딩 기록 보기
- 점수 입력/수정
- 공유 링크 점수 입력
- 순위표
- 개인점수관리
- 구장 관리

## 2. 구현 결과

### 기능 수정

- 라운딩 기록 삭제 후 Supabase 폴링으로 삭제된 기록이 다시 보이는 문제를 방지했습니다.
- 삭제된 라운딩 ID를 클라이언트 차단 목록으로 유지하고, 원격 데이터를 다시 불러올 때 해당 ID를 필터링합니다.
- CSV 내보내기와 CSV 양식 저장은 지원 브라우저에서 파일 저장 대화창을 열도록 변경했습니다.
- 지원하지 않는 브라우저에서는 기존 다운로드 방식으로 fallback됩니다.
- 경기방식에 `청백전`을 추가하고, 팀 단위 점수 집계 대상으로 처리했습니다.
- 기준파 입력은 코스별 1~9홀 표 형태로 정리했습니다.
- 코스별 기준파 합계 36타 검증은 입력 중 실시간으로 표시되며, 별도 팝업을 띄우지 않습니다.
- 점수 입력/수정과 공유 링크 점수 입력 모두 같은 코스 탭형 그리드 UI를 사용합니다.
- 점수 셀에는 `홀인원`, `알바트로스`, `이글`, `버디`, `파` 등 성과 라벨을 표시합니다.
- 공유 링크 점수 입력 화면에서도 상단에서 입력 대상 또는 조를 바꿀 수 있습니다.

### UI/UX 수정

- 홈 문구 `오늘의 라운드가, / 나의 기록이 된다.`를 항상 두 줄로 표시합니다.
- 홈 보조 문구는 반응형에서도 한 줄로 유지되도록 크기와 배치를 조정했습니다.
- 다크모드에서 등록 회원 숫자, 회원 카드, 스코어 표 보조 텍스트가 묻히는 문제를 보정했습니다.
- 회원 확장 카드의 수정/삭제 버튼은 모바일에서 요약 영역 아래 가로 버튼으로 배치했습니다.
- 라운딩 생성과 참가자 선택에 상단 고정형 단계 진행바를 추가했습니다.
- 브라우저 뒤로가기는 라운딩 생성/참가자 선택의 바로 이전 단계로 이동하도록 히스토리 상태를 세분화했습니다.
- 중복되는 `기본 정보 수정`, `편성 방식 수정`, `참가자 수정` 버튼은 제거했습니다.
- 점수 입력 화면은 코스 탭, 축소 토글, 좌우 스와이프, 콤팩트 그리드 구조를 적용했습니다.
- 카드와 글자 크기를 줄이고 핵심 숫자 중심의 스포츠 플랫폼 톤을 유지했습니다.

## 4. 디자인/UX 영역 분석

### High Quick Win

문제:

- 전체 그리드 점수 입력은 편하지만, 보조 텍스트가 많으면 모바일에서 운영표처럼 보일 위험이 있습니다.

해결:

- 코스 탭은 유지하고, 한 코스 9홀만 한눈에 보이도록 구성했습니다.
- 각 셀은 홀 번호, 파, 입력값, 성과 라벨만 남겼습니다.
- 숫자 대비와 터치 영역은 44px 이상을 유지했습니다.

추가 권장:

- 점수 입력 화면 상단에 `저장됨`, `동기화 중`, `오프라인` 상태 배지를 추가하면 현장 신뢰도가 높아집니다.

### High Quick Win

문제:

- 다크모드에서 상태 색상과 일반 텍스트 색상이 충돌하면 야간 라운딩 중 사용성이 크게 떨어집니다.

해결:

- 다크모드 카드, 회원 요약, 스코어 표, 버튼, 보조 텍스트 색상을 고대비로 재정의했습니다.
- 기준파 대비 `0`처럼 글로벌 상태 클래스가 어둡게 보이는 문제를 별도 보정했습니다.

추가 권장:

- 사용자 선택형 다크모드 토글을 추가하고, `prefers-color-scheme` 자동 모드와 함께 제공하는 것이 좋습니다.

### Medium Quick Win

문제:

- 라운딩 생성은 입력 항목이 많아 사용자가 현재 위치를 잃기 쉽습니다.

해결:

- 기본 정보, 경기 설정, 참가자, 편성 방식, 조 인원 흐름을 단계 진행바로 분리했습니다.
- 뒤로가기는 바로 전 단계로 이동하도록 보정했습니다.

추가 권장:

- 단계별 완료 조건을 진행바에 체크 아이콘으로 표시하면 불안감이 더 줄어듭니다.

### Medium 장기 과제

문제:

- 공유 카드가 아직 스포츠 플랫폼의 바이럴 자산으로 충분히 강하지 않습니다.

해결 방향:

- 공유 카드에는 라운딩명, 장소, 날짜, 우승자, 기준파 대비 성적, 상위 3명, 개인 배지, 동호회명을 반드시 포함해야 합니다.
- `html-to-image`, `satori`, `sharp` 조합을 검토하면 DOM 기반 템플릿 품질을 높일 수 있습니다.

### Low 장기 과제

문제:

- 현재 모션은 CSS 기반입니다. 사용자가 요청한 Framer Motion 수준의 전환 설계까지는 아직 가지 않았습니다.

해결 방향:

- 현재 프로젝트는 Next.js가 아니라 Vite React입니다.
- Framer Motion 도입은 가능하지만, 현재 로컬 npm 경로가 없어 패키지 추가를 수행하지 못했습니다.
- 장기적으로 `framer-motion`을 추가해 화면 전환, 슬라이드 업 모달, 스와이프 탭 전환을 컴포넌트 단위로 정리하는 것이 좋습니다.

## 5. 기술/운영 영역 분석

### High Quick Win

문제:

- 라운딩 삭제 직후 원격 폴링이 예전 데이터를 다시 덮어쓰면 삭제한 기록이 부활할 수 있습니다.

해결:

- 삭제 ID 차단 목록을 두어 원격 데이터 로드 시 삭제된 기록을 필터링했습니다.
- 다음 저장 주기에서 Supabase 정규 테이블의 누락 row 삭제가 반영됩니다.

추가 권장:

- 장기적으로는 `deleted_at` soft delete와 서버 기준 동기화 버전을 두는 방식이 더 안전합니다.

### High Quick Win

문제:

- 여러 사용자가 같은 입력 대상의 점수를 동시에 수정하면 마지막 저장이 이전 값을 덮어쓸 수 있습니다.

해결 방향:

- `parkbuddy_round_scores`에 `version`, `updated_at`, `updated_by`를 두고 optimistic locking을 적용해야 합니다.
- 저장 시 버전 충돌이 나면 "다른 사용자가 방금 수정했습니다" 안내와 병합 선택지를 제공해야 합니다.

### High 장기 과제

문제:

- 여러 동호회를 운영하려면 현재 anon 기반 접근은 보안상 한계가 있습니다.

해결 방향:

- Supabase Auth를 도입하고 `club_memberships(user_id, club_id, role)` 기준 RLS를 적용해야 합니다.
- 관리자는 모든 동호회, 일반 사용자는 가입 동호회만 조회하도록 정책을 분리해야 합니다.

### Medium Quick Win

문제:

- 모바일 네트워크가 불안정하면 실시간 점수 저장 실패가 늦게 드러날 수 있습니다.

해결 방향:

- 저장 실패 시 셀 단위 재시도 표시, 마지막 저장 시간, 재전송 버튼을 추가합니다.
- 권장 라이브러리: TanStack Query mutation retry, `idb-keyval` 기반 임시 저장 큐.

### Medium 장기 과제

문제:

- 현재 실시간 순위는 주기적 동기화 중심이라 동시 접속자가 늘면 REST polling 비용이 커집니다.

해결 방향:

- Supabase Realtime으로 라운딩별 채널을 분리합니다.
- 점수 row 변경 이벤트만 구독하고, 순위 계산은 클라이언트 또는 Edge Function 캐시로 분리합니다.

### Low 장기 과제

문제:

- React Router가 없어 화면별 URL 복원과 새로고침 UX가 제한적입니다.

해결 방향:

- `react-router-dom`을 도입해 `/records`, `/records/:recordId`, `/score/:recordId/:entryId` 구조로 전환합니다.

## 6. 우선순위 개선 로드맵

High:

- Supabase Auth + RLS + 동호회 멤버십 권한 적용
- 점수 저장 optimistic locking
- 삭제 기록 soft delete와 서버 동기화 버전 관리
- 모바일 저장 실패 재시도/오프라인 큐

Medium:

- Supabase Realtime 순위 push 갱신
- Framer Motion 도입 또는 CSS View Transition 기반 화면 전환 정리
- 공유 카드 DOM 템플릿화
- 카카오 공유 SDK와 문자 발송 API 연동
- 라운딩 중 참가자 제외, 조 이동, 코스 변경 기능

Low:

- 다크모드 수동 토글
- 개인 성장 데이터에 코스별 버디율/파세이브율 추가
- 운영자용 PDF 리포트
- 구장별 난이도 데이터 축적

## 7. 결론

이번 수정으로 삭제 안정성, 단계형 라운딩 생성, 실시간 기준파 검증, 공유 링크 점수 입력, 다크모드 가독성, 모바일 점수 입력 밀도가 개선되었습니다. 현재 앱은 동호회 운영 도구를 넘어 스포츠 데이터 플랫폼의 형태에 가까워졌습니다.

남은 핵심 과제는 인증/권한, 실시간 충돌 방지, 오프라인 저장 방어, Realtime 기반 순위 갱신입니다. 이 네 가지를 보강하면 여러 동호회를 동시에 운영하는 구조로 확장할 수 있습니다.

# ParkBuddy Web 평가분석보고서

작성일: 2026-06-04

## 1. 평가 범위

이번 평가는 개선 포인트와 계획된 MVP 개발 항목의 구현 완료 여부, 주요 사용자 흐름의 정상 동작 여부, 배포 안정성을 대상으로 수행했습니다.

평가 대상 기능:

- 회원 등록/수정/삭제/검색, CSV 일괄등록
- 동호회 이름/직책 기반 회원 정보 관리
- 사용자 구장 등록/삭제와 라운딩 생성 연동
- 홀별 규정타수 입력 및 점수 입력 기본값 반영
- 신페리오 숨김 홀 자동 선정, 직접 선택, 공개 여부 설정
- 스트로크 플레이, 신페리오, 매치 플레이, 스크램블, 포섬, 포볼, 스테이블포드 순위 계산
- 조편성, 팀전/페어전 점수 입력, 기록 저장
- 라운딩 기록 상세 보기, CSV 내보내기, 삭제
- Supabase 자동 저장
- GitHub Pages/로컬 Vite 실행 안정성

## 2. 구현 완료 평가

기능 구현 상태는 MVP 기준으로 완료로 평가합니다.

- 회원 관리: 검색, 소속 동호회, 직책, 연락처 형식, CSV 인코딩/파싱 안정성이 반영되었습니다.
- 경기 운영: 라운딩 생성부터 참가자 선택, 조편성, 점수 입력, 순위표, 기록 저장까지 단일 흐름으로 동작합니다.
- 경기 방식: 개인전, 팀전, 페어전, 포인트 기반 경기 방식을 모두 같은 순위 엔진에서 처리합니다.
- 신페리오: 홀 규모별 2/3 숨김 홀 원칙을 9홀 블록 단위로 적용하고, 파 합계도 전체 기준파의 2/3에 맞추도록 설계되었습니다.
- 기록 관리: 저장된 기록의 상세 확인, 삭제, CSV 내보내기가 가능해졌습니다.
- 저장 방식: Supabase 단일 JSON 상태 저장에 회원, 기록, 최근 장소, 사용자 구장을 포함했습니다.

## 3. 검증 결과

빌드 검증:

```text
npm.cmd run build
결과: 통과
```

자동 브라우저 검증:

```text
C:\Program Files\nodejs\node.exe dev\capture-flow.mjs
결과: 통과
캡처 경로: C:\Capture
```

검증된 화면 흐름:

- 홈
- 회원 관리
- 라운딩 생성
- 참가자 선택
- 조편성 결과
- 점수 입력
- 순위표
- 기록 목록/상세
- 구장 관리 추가/삭제
- 신페리오, 매치 플레이, 스크램블, 포섬, 포볼, 스테이블포드별 주요 라운딩 흐름

대표 스크린샷을 확인한 결과, 주요 화면의 텍스트 겹침이나 blank 렌더링 문제는 발견되지 않았습니다.

## 4. 품질 분석

강점:

- 하나의 점수 입력/순위 계산 구조로 개인전과 팀전을 함께 처리해 확장성이 좋아졌습니다.
- 신페리오 숨김 홀 선정은 9홀 단위 배분, 숨김 홀 수, 파 합계 기준을 함께 고려합니다.
- Supabase 저장 대상을 확장하면서도 기존 단일 상태 저장 구조를 유지해 변경 위험을 줄였습니다.
- 자동 브라우저 스크립트가 전체 핵심 흐름을 반복 검증할 수 있게 되었습니다.

제한 사항:

- 로그인과 권한 관리는 아직 구현되지 않았습니다.
- Supabase는 MVP 단계의 단일 JSON 상태 저장 방식이므로, 다중 동호회 운영이 본격화되면 정규 테이블과 RLS 정책으로 분리해야 합니다.
- 모든 참가자의 점수가 동일하면 공동 순위로 표시됩니다. 운영 규칙상 순위를 강제로 나눠야 한다면 추가 타이브레이커가 필요합니다.
- CSV 내보내기는 기록 단위 수동 다운로드 방식이며, 기간별 통계 리포트는 아직 없습니다.

## 5. 결론

현재 ParkBuddy Web은 계획된 MVP 운영 흐름을 웹앱으로 수행할 수 있는 상태입니다. 회원 관리, 라운딩 생성, 조편성, 점수 입력, 순위 계산, 기록 관리, Supabase 저장, 자동 검증이 모두 연결되었습니다.

다음 단계의 우선순위는 로그인 도입, 동호회별 권한 분리, Supabase 정규 테이블 전환, 관리자/일반 사용자 화면 분기입니다.
